1 Sankey Diagrams

1.1 What is a Sankey Diagram

A Sankey diagram is a specific type of flow chart where branch widths are proportional to the flow quantity, used to visualize the transfer of energy, money, or materials. It is named after Matthew Henry Phineas Riall Sankey’s 1898 diagram of steam engine energy efficiency (Wikipedia).

A Sankey diagram consists of nodes and links. Nodes represent stages or entities in a process; links represent the flow of resources between nodes, typically directed from a source node to a target node. The width of a link represents the flow quantity, and colors can distinguish different stages or categories. Flow is conserved: the sum of branch widths at each stage is equal.

Example 1: The figure below (Figure 1) shows the flow of electricity in a home: total power on the far left, rooms in the middle, and appliances on the right; link width represents flow quantity, and colors distinguish appliance types.

(Image source: PPCexpo, Example #2)

Example 2: The figure below (Figure 2) shows the classification of orders on a shopping website: starting from total orders, broken down by product category, subcategory, and brand; link width represents order quantity, and colors distinguish product categories.

(Image source: ChartExpo, Example #5)

1.4 Dictionaries

A dictionary (dict) is a Python data structure that stores data as key-value pairs. There are three common ways to create one:

# Method 1
d1 = {'a': 1, 'b': 2, 'c': 3}  # keys must be quoted

# Method 2
d2 = dict(a=1, b=2, c=3)  # no quotes needed

# Method 3
keys = ['a', 'b', 'c']  # define keys and values separately
values = [1, 2, 3]
d3 = dict(zip(keys, values))  # zip() pairs them together, dict() converts to a dictionary

print(d1)
print(d2)
print(d3)
{'a': 1, 'b': 2, 'c': 3}
{'a': 1, 'b': 2, 'c': 3}
{'a': 1, 'b': 2, 'c': 3}

Key differences between lists and dictionaries:

  • Lists are ordered, elements can repeat, and elements are accessed by position (index, starting from 0).
  • Dictionaries are unordered, keys must be unique (values can repeat), and values are accessed by key.
# Example: animals and their sounds
animal_with_sound = {'cat': 'meow', 'dog': 'woof', 'duck': 'quack'}
sound = ['meow', 'woof', 'quack']
# List: access elements by position (index)
sound[0]
'meow'
# Dict: access values by key (note: dicts are unordered, cannot use index)
animal_with_sound['cat']
'meow'

1.5.1 Building the Sankey Object

Step 1: Use go.Sankey() to create the Sankey diagram object. In the node names, A/B/C represent three stages, and numbers represent different nodes within the same stage.

# Parameter configuration for go.Sankey()

sankey_data = go.Sankey(   # define the sankey_data variable

    ##########################################################################
    # Node
    node = dict(

        # Node labels
        label = ['A1','A2','B1','B2','C1','C2'],

        # Node colors: one-to-one correspondence with node labels
        color = [
            'yellow', 'yellow', 'tomato', 'tomato',
            'cornflowerblue', 'cornflowerblue',
        ],

        # Node spacing (padding) and thickness (unit: px)
        pad = 30,
        thickness = 20,

        # Node border
        line = dict(
            color = 'black', # border color
            width = 5,       # border width (unit: px)
        )
    ),

    ##########################################################################
    # Link
    link = dict(

        # Source nodes: index in the node label list
        source = [0, 1, 0, 2, 3, 3],

        # Target nodes: index in the node label list
        target = [2, 3, 3, 4, 4, 5],

        # Link flow values: corresponding to source and target lists
        value = [8, 4, 2, 8, 4, 2],

        # Link colors: corresponding to the above
        color = [
            'sandybrown', 'sandybrown', 'sandybrown',
            'thistle', 'thistle', 'thistle',
        ],

        # Link arrow length: pointing from source to target (default 0, no arrow)
        arrowlen = 15
    )
)

##############################################################################
# Check the data type of sankey_data
print('Data type:\n', type(sankey_data),'\n')

# Check the content of sankey_data
print('Content:\n', sankey_data)
Data type:
 <class 'plotly.graph_objs._sankey.Sankey'> 

Content:
 Sankey({
    'link': {'arrowlen': 15,
             'color': [sandybrown, sandybrown, sandybrown, thistle, thistle,
                       thistle],
             'source': [0, 1, 0, 2, 3, 3],
             'target': [2, 3, 3, 4, 4, 5],
             'value': [8, 4, 2, 8, 4, 2]},
    'node': {'color': [yellow, yellow, tomato, tomato, cornflowerblue,
                       cornflowerblue],
             'label': [A1, A2, B1, B2, C1, C2],
             'line': {'color': 'black', 'width': 5},
             'pad': 30,
             'thickness': 20}
})

As seen from the output, sankey_data is a Plotly Sankey object, with parameters stored in nested dictionaries.

When building a Sankey object, note that values in the source and target node lists are indices into the node label list, not node names themselves. For example, 0 in source = [0, 1, 0, 2, 3, 3] refers to the first node ‘A1’, 1 refers to the second node ‘A2’, and so on. Therefore, you must ensure that indices match the order of nodes in the label list, otherwise links will point to the wrong nodes.

1.5.3 Pre-defining Variables

As the number of nodes grows, managing indices becomes difficult. To address this, we can define a dictionary to look up the index of a node by its name, making the data construction more intuitive:

# Define a list variable to store node names
node_names = ['A1', 'A2', 'B1', 'B2', 'C1', 'C2']

# Generate index list corresponding to nodes using range(len())
node_ids = range(len(node_names))

# Build a name-to-index dictionary
nodes = dict(zip(node_names, node_ids))
nodes['B2']  # look up the index for a node by name
3

This allows you to look up the index of source nodes and target nodes in the node label list by name.

At the same time, for long parameters, you can pre-define variables and pass them into the function, making the code cleaner and easier to maintain. The following code redraws the same Sankey diagram shown in Figure 3 above:

##############################################################################
# Values needed for node parameters

# Define node colors
node_colors = [
    'yellow', 'yellow', 'tomato', 'tomato',
    'cornflowerblue', 'cornflowerblue',
]

# Define the complete node parameter
node_definition = dict(
    pad=15,
    thickness=20,
    line=dict(color='black', width=5),
    label=node_names,   # use the node name variable defined above
    color=node_colors,  # use the node color variable just defined
)

##############################################################################
# Values needed for link parameters

# Look up indices by node name to define sources and targets
sources = [
    nodes['A1'], nodes['A1'], nodes['A2'],
    nodes['B1'], nodes['B2'], nodes['B2'],
]
targets = [
    nodes['B1'], nodes['B2'], nodes['B2'],
    nodes['C1'], nodes['C1'], nodes['C2'],
]

# Define link flow values and colors
values = [8, 4, 2, 8, 4, 2]
link_colors = [
    'sandybrown', 'sandybrown', 'sandybrown',
    'thistle', 'thistle', 'thistle',
]

# Define the complete link parameter
link_definition = dict(
    source=sources,
    target=targets,
    value=values,
    color=link_colors,
    arrowlen=15,
)

##############################################################################
# Draw the Sankey diagram

# Pass the pre-defined node_definition and link_definition into the function
fig = go.Figure(
    go.Sankey(
        node = node_definition,
        link = link_definition,
    )
)

# Update layout
fig.update_layout(
    height=400,
    title_text='Sample Sankey Diagram - Redrawn',
    title_x=0.5, title_y=0.95,
    margin=dict(
        l=60, r=60, t=60, b=20,
    ),
    font={'style': 'italic'},
)
fig.show() # display the chart

The redrawn result is identical to the original. But by pre-defining data as variables, you only need to adjust variable values when modifying data, making the code cleaner and easier to maintain.

1.5.4 Modifying Color Transparency

To modify link color transparency, use RGBA values instead of color name strings. The RGBA format is (red, green, blue, alpha), where alpha represents opacity (0 = fully transparent, 1 = fully opaque). Both "sandybrown" and "thistle" are CSS preset color names, which can be found at w3schools CSS Colors or the Matplotlib Color Reference.

# Import library for handling RGBA values
import matplotlib.colors as c
# Use c.to_rgba() to find RGB values and set alpha
c.to_rgba('sandybrown', 0.5)
(0.9568627450980393, 0.6431372549019608, 0.3764705882352941, 0.5)

Plotly supports specifying RGBA colors using the string format 'rgba(r, g, b, a)'.

# Define strings to store RGBA values for transparent colors
faint_sandybrown = 'rgba' + str(c.to_rgba('sandybrown', 0.5)) # + concatenates strings
faint_thistle = 'rgba' + str(c.to_rgba('thistle', 0.5))

# Print the strings
print(faint_sandybrown)
print(faint_thistle)
rgba(0.9568627450980393, 0.6431372549019608, 0.3764705882352941, 0.5)
rgba(0.8470588235294118, 0.7490196078431373, 0.8470588235294118, 0.5)
# Redefine link colors
link_colors = [
    faint_sandybrown, faint_sandybrown, faint_sandybrown,
    faint_thistle, faint_thistle, faint_thistle,
]

# Update chart
fig.update_traces(link_color=link_colors)

fig.update_layout(
    height=400,
    title_text='Sample Sankey Diagram - 50% Link Transparency',
    title_x=0.5, title_y=0.95,
    margin=dict(
        l=60, r=60, t=60, b=20,
    ),
    font={'style': 'italic'},
)
fig.show()

# Import libraries for drawing network graphs
from pyvis.network import Network
import networkx as nx

Step 1: Create network relationships with networkx

# Create an empty undirected graph
gx = nx.Graph()
# Note: use nx.MultiDiGraph() to create a directed graph instead
# Add nodes and edges to the empty graph gx
gx.add_edge('a', 'b')  # edge from 'a' to 'b'
gx.add_edge('b', 'd')

gx.add_edge('e', 'a')

# Create an empty interactive network graph object
gp = Network()

# Load network relationships from gx

2.3 Reading Network Relationships from a File

In addition to manual creation, network relationship data can also be read from a CSV file. The following example uses beer and curry tweet data (view source file): original posters mentioned other users, forming directed edges from the original poster (source node) to the mentioned user (target node).

###############################################################################
# Data preprocessing

# Import pandas to handle the CSV file
import pandas as pd

# Read the CSV file
path = '../../../src/data/beer-and-curry.csv'
df = pd.read_csv(path)

# Drop rows with no mentions
df.dropna(subset='mentions', inplace=True)

###############################################################################
# Create network relationships

# Create an empty directed graph
gx1 = nx.MultiDiGraph()

# Loop over all rows
for i in df.index:
    user = df.loc[i, 'user']              # get the user as source node
    mentions = df.loc[i, 'mentions']      # get all mentions in a post
    mentions = mentions.replace(' ', '')  # remove spaces
    mentions = mentions.split('@')        # split by '@' to get individual mentions
    mentions.pop(0)                       # first element after split is empty, remove it

    for mention in mentions:
        gx1.add_edge(user, mention)       # add all nodes and edges

###############################################################################
# Draw the network graph

gp1 = Network()   # create an empty network graph
gp1.from_nx(gx1)  # load network relationships
gp1.show('net-from-csv.html', notebook=False)  # output HTML file

The output network graph is complex; rendering may take 1–2 minutes — do not close the browser during this time. The overall view is shown in Figure 9, and a detail view in Figure 10.

Network graphs can be used to study social networks, identify strong/weak connections and high-influence users, and analyze potential information cocoon or echo chamber phenomena.

```{python}
    line=dict(color='black', width=5),
    label=node_names,   # use the node name variable defined above
    color=node_colors,  # use the node color variable just defined
)

##############################################################################
# Values needed for link parameters

# Look up indices by node name to define sources and targets
sources = [
    nodes['A1'], nodes['A1'], nodes['A2'],
    nodes['B1'], nodes['B2'], nodes['B2'],
]
targets = [
    nodes['B1'], nodes['B2'], nodes['B2'],
    nodes['C1'], nodes['C1'], nodes['C2'],
]

# Define link flow values and colors
values = [8, 4, 2, 8, 4, 2]
link_colors = [
    'sandybrown', 'sandybrown', 'sandybrown',
    'thistle', 'thistle', 'thistle',
]

# Define the complete link parameter
link_definition = dict(
    source=sources,
    target=targets,
    value=values,
    color=link_colors,
    arrowlen=15,
)

##############################################################################
# Draw the Sankey diagram

# Pass the pre-defined node_definition and link_definition into the function
fig = go.Figure(
    go.Sankey(
        node = node_definition,
        link = link_definition,
    )
)

# Update layout
fig.update_layout(
    height=400,
    title_text='Sample Sankey Diagram - Redrawn',
    title_x=0.5, title_y=0.95,
    margin=dict(
        l=60, r=60, t=60, b=20,
    ),
    font={'style': 'italic'},
)
fig.show() # display the chart
```

The redrawn result is identical to the original. But by pre-defining data as variables, you only need to adjust variable values when modifying data, making the code cleaner and easier to maintain.

### Modifying Color Transparency

To **modify link color transparency**, use RGBA values instead of color name strings. The RGBA format is `(red, green, blue, alpha)`, where `alpha` represents opacity (0 = fully transparent, 1 = fully opaque). Both `"sandybrown"` and `"thistle"` are CSS preset color names, which can be found at [w3schools CSS Colors](https://www.w3schools.com/cssref/css_colors.php) or the [Matplotlib Color Reference](https://matplotlib.org/stable/gallery/color/named_colors.html).

```{python}
# Import library for handling RGBA values
import matplotlib.colors as c
```

```{python}
# Use c.to_rgba() to find RGB values and set alpha
c.to_rgba('sandybrown', 0.5)
```

Plotly supports specifying RGBA colors using the string format `'rgba(r, g, b, a)'`.

```{python}
# Define strings to store RGBA values for transparent colors
faint_sandybrown = 'rgba' + str(c.to_rgba('sandybrown', 0.5)) # + concatenates strings
faint_thistle = 'rgba' + str(c.to_rgba('thistle', 0.5))

# Print the strings
print(faint_sandybrown)
print(faint_thistle)
```

```{python}
#| label: fig-sankey-sample-faint
#| fig-cap: "Sample Sankey Diagram - 50% Link Transparency"
# Redefine link colors
link_colors = [
    faint_sandybrown, faint_sandybrown, faint_sandybrown,
    faint_thistle, faint_thistle, faint_thistle,
]

# Update chart
fig.update_traces(link_color=link_colors)

fig.update_layout(
    height=400,
    title_text='Sample Sankey Diagram - 50% Link Transparency',
    title_x=0.5, title_y=0.95,
    margin=dict(
        l=60, r=60, t=60, b=20,
    ),
    font={'style': 'italic'},
)
fig.show()
```



# Network Graphs

## Comparing Sankey Diagrams and Network Graphs

Both Sankey diagrams and network graphs visualize relational data, but with different emphases, as shown in @tbl-sankey-network:

| Feature | Sankey Diagram | Network Graph |
|:------:|:--------:|:--------:|
| Primary Use | Visualize resource flow quantities | Visualize connections between nodes |
| Node Meaning | Process stage or entity | Individual (e.g., a user) |
| Connection Name | Link | Edge |
| Connection Direction | Directed (source → target) | Can be directed or undirected |
| Connection Width | Represents flow quantity | No width distinction |
| Key Characteristics | Hierarchical, flow conservation | Connection strength and relational distance |
| Application Examples | Energy distribution, cost allocation | Social networks, computer networks |

: Comparison of Sankey Diagrams and Network Graphs {#tbl-sankey-network}

## Reading Network Relationships from a File

In addition to manual creation, network relationship data can also be read from a CSV file. The following example uses ***beer and curry*** tweet data [(view source file)](https://github.com/Pinn32/dv-projects/blob/main/src/data/beer-and-curry.csv): original posters mentioned other users, forming directed edges from the original poster (source node) to the mentioned user (target node).

```{python}
#| output: false
###############################################################################
# Data preprocessing

# Import pandas to handle the CSV file
import pandas as pd

# Read the CSV file
path = '../../../src/data/beer-and-curry.csv'
df = pd.read_csv(path)

# Drop rows with no mentions
df.dropna(subset='mentions', inplace=True)

###############################################################################
# Create network relationships

# Create an empty directed graph
gx1 = nx.MultiDiGraph()

# Loop over all rows
for i in df.index:
    user = df.loc[i, 'user']              # get the user as source node
    mentions = df.loc[i, 'mentions']      # get all mentions in a post
    mentions = mentions.replace(' ', '')  # remove spaces
    mentions = mentions.split('@')        # split by '@' to get individual mentions
    mentions.pop(0)                       # first element after split is empty, remove it

    for mention in mentions:
        gx1.add_edge(user, mention)       # add all nodes and edges

###############################################################################
# Draw the network graph

gp1 = Network()   # create an empty network graph
gp1.from_nx(gx1)  # load network relationships
gp1.show('net-from-csv.html', notebook=False)  # output HTML file
```

The output network graph is complex; rendering may take 1–2 minutes — do not close the browser during this time. The overall view is shown in @fig-network-overview, and a detail view in @fig-network-detail.

![Social network graph: overall view](../../../src/img/ttl/sankey/fig9-network-overview.png){#fig-network-overview}

![Social network graph: detail view](../../../src/img/ttl/sankey/fig10-network-detail.png){#fig-network-detail}

Network graphs can be used to study social networks, identify strong/weak connections and high-influence users, and analyze potential information cocoon or echo chamber phenomena.
